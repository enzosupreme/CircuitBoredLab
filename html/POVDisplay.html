POV Display

A POV Display is persistence of vision display. It is composed of a row of lights that blink in a pattern that creates an optical illusion. I alwayys thought they looked cool so I wanted to make one and practice some more circuit python.

Stage 1: Design and Build

I designed this with interactivity in mind. I wanted to make a POV display that my niece would play with. I wanted something hand powered but still worked as a POV display. The first design was going to be a remix of the penquin zoetrope found on thingiverse. After researching how they work I found the best way to make a POV display involved using a hall sensor(a sensor that detects magnets) and magnets to obtain location of LEDs while they rotate. I had to find some tutorials to figure out how to redesign the base and make a version with an embedded magnet. I was able to pull it off, but the design made it dificult to rotate the LEDs consistently. I came up with a design that used a ball bearing that you flick and keeps a good rotation for a short time until you manually add more momentum.

Stage 2: Code

First I had to look up the documentation on the neopixel rgb lights I used for this project. After learning how to control the LEDs, I came up with a basic loop that cycles through a library of colors I setup. I created an alphabet "library" by making an list of values per letter of the alphabet and numebers 0-9. Each character contain a list of values of either 0 or x, x being the current color. Either value gets fed as an argument for a function I wrote that turns the LEDs off and on in sequence. For the timing I borrowed a formula from another POV display project that used arduino. After converting the formula into circuit python I was able to properly time the lights using the hall sensor to calcuate a time of rotation. Using the function I wrote, I called the function using the letters of my niece's first name as arguments to spell her name on the POV display.
