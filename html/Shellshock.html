Shell Shock

I decided to build a Redox split key keyboard using KMK firmware and Circuit Python. The Color Scheme was loosly Inspired by TMNT.

Stage 1: Design and Build

I didn't need to design anything for this build I printed the enclosure straight from thingiverse. Instead of ordering a custom PCB, i decided to handwire the keyboard. It isn't too difficult to do, but kind of time consuming. The over all process is somewhat therapuetic though. First I started by prepping the diodes by bending one end and clipping the excesss. I had to make one for each key, 75 total.

I printed the base of the enclosure in olive drab and printed the top with wood PLA. It is plastic filament with a small percentage of wood. Its not very convincing fresh off of the printer, but the material can be sanded and stained, to make it look more authentic.

After wiring both sides up to a microcontroller it was time to flash the software.

Stage 2: Code

I went to the kmk firmware github page to get started. I went over some of the example default layouts to get started on adapting the redox to circuit python. After a lot of tinkering and reading over the kmk documentation, I was able to come up with a base layer keymap. Both sides of the keyboard worked independently when plugged in after loading the software to the boards. For some reason the side not plugged in with the USB cable would not work when plugged into the other side. I struggled with troubleshooting and researching a fix as well as rewiring multiple components in my build. I could not get both sides working in tandem. I eventually resorted to hardwiring the right side to the left side using a pi pico which provided enough pins to pull this off.

After rewiring the keyboard and make some last modifications to the code, everything worked as intended. It was now time to iron out the layout.

Stage 3: Layout

I went with a three layer layout for this build. they first layer is mostly just a standard qwerty layout. I setup two backspaces in the center where they are easy to reach since I use them so much. I have the four arrow keys, parentheses and brackets all in the center. Lastly in the center are the two space bar keys. I like have the shift key there as well, so I set it up to be space bars on tap and shift keys on hold. For the second layer, the right side becomes the keypad giving my keyboard a built in tenkey without the extra physical keys. For the left side, WASD becomes the arrow keys for easy navigation. Underneath, I swapp the first layer keys with their corresponding edit functions(undo,cut,copy,paste). The up and down arrows in the center become the volume controls and the right and left arrows can be used to change tracks. I use the third layer as an experimental layer. I set up the left side as a launch pad. I have keys set to launch four of my favorite programs. On the right side I made shortcuts for emotes and features for Discord. I plan to keep evolving this keyboard as I go along. I was kind of overwhelmed with the possiblities this firmware allows, so from here I'll be adding features as they come to mind.
