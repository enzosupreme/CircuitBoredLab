GM Shark

This didn't start out as another dice roller. I made this as a DM tool for table top games. The GM Shark contains a weapon generator and custom dice to help with DMing. The GM Shark uses a rotary encoder and two keyswitches. The whole thing was written in Circuit Python. The goal was to build skills in writing circuit python and to implement my lootbox idea into a physical device.

Stage 1: Design and Build

Having gained some skills with Fusion360 while working on the second diceboard, I felt comfortable enough to experiment a bit with a handheld device design. I used adobe illustrator to create an SVG file of the text to use in F360 to engrave the enclosure.

I wanted to use a rotary encoder to cycle through options and have two buttons with separate functions. I also made it battery powered using a Lipo battery.

Stage 2: Code

The first callenge was getting te OLED display working via circuit python. I used the adafruit website and the official documentation to figure out how to display text to the display.

After getting the display to display text, getting the buttons working then the encoder. After figuring out both, I wrote some test code to send text to the display after key presses and position changes. One roadblock I ran into was printed intergers to the display. I resorted to converting all integers to strings just to get the right text to display. I set the encoder to cycle through the number of sides for each die. I setup the right button to roll the current selected die and the left button to print the total of the accumulated dice rolls. I then set out to solve implementing the weapon generator. For this I flipped through the 5th edition of the Dungeons and Dragons Player's handbook to see what data I needed to make weapons. Since everything is text based I just needed the data to represent the values of the required stats for each weapon. Each weapon needed a name(or type),Level, Damage Type, Damage, Cost(or price), and weight. I knew I could write a function to get random numbers for most of those, but determining damage type wouldn't be as straight forward. Luckily there are only three damage types, so I put them in a list as strings then organized the list of weapon names in order. I then made a condition that checks weapon name first then assigns the appropriate damage type based on the name's position in the list. After that it was just a matter of formatting the layout of the text and adding that function to the encoder's position change.
